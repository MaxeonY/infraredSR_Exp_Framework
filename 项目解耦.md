# 项目框架与 M3FD 数据集解耦报告

## 目标

本次改造将项目的数据集层从 M3FD 专用实现调整为通用灰度图像超分辨率数据集接口。M3FD 仍作为内置示例和历史兼容数据集保留，但训练、测试和预处理主流程不再直接依赖 `M3FDSRDataset`、`build_m3fd_dataset` 或 `configs/dataset/m3fd.yaml`。

## 实施结果

### 1. 通用 Dataset 抽象

新增 `datasets/sr_dataset.py`，定义 `GenericSRDataset`。

关键能力：

- 支持灰度图像 SR 训练、验证和测试。
- 支持 `raw_root` 参数，用于 split 路径 remap。
- 保留原有 degradation、随机裁剪、确定性评估、缓存加载逻辑。
- 支持历史 M3FD split 路径兼容 remap。

路径解析顺序：

1. split 行为绝对路径且存在，直接使用。
2. split 行为相对路径时，尝试 `project_root / line`。
3. 如果传入 `raw_root`，尝试 `raw_root / line`。
4. 对包含历史 `data/raw/m3fd` 片段的路径，尝试 remap 到 `raw_root` 或兼容默认 M3FD 根目录。
5. 仍找不到则跳过并输出 warning。

### 2. 兼容层保留

保留以下旧接口，避免旧代码立即失效：

- `datasets/m3fd_dataset.py` 中保留 `M3FDSRDataset = GenericSRDataset`。
- `datasets/builder.py` 中保留 `build_m3fd_dataset = build_sr_dataset`。
- `datasets/kaist_dataset.py` 中保留 `KAISTSRDataset = GenericSRDataset`。

### 3. Builder 和包导出泛化

`datasets/builder.py` 新增 `build_sr_dataset()`。

`datasets/__init__.py` 导出：

- `GenericSRDataset`
- `M3FDSRDataset`
- `build_sr_dataset`
- `build_m3fd_dataset`

### 4. 数据集注册表

新增 `datasets/registry.py`。

当前内置注册项：

- `m3fd`: `data/raw/M3FD`, `Ir`
- `kaist`: `data/raw/KAIST`, 空子目录

注册表只作为默认值来源；用户仍可通过 `--raw-root` / `--raw-data-root` 直接接入任意数据集，不强制注册。

### 5. 通用数据集配置

新增 `configs/dataset/default.yaml`。

该配置只包含通用 degradation 参数，不写死 `dataset_name: m3fd` 或 `raw_data_root: data/raw/M3FD`。

`configs/dataset/m3fd.yaml` 保留，用于旧实验和显式 M3FD 配置兼容。

### 6. 预处理流程泛化

`datasets/preprocess.py` 已从 M3FD 专用脚本调整为通用图像 split 生成脚本。

新增参数：

- `--dataset_name` / `--dataset-name`
- `--raw-root` / `--raw_root`
- `--image-subdir` / `--image_subdir`
- `--processed-root` / `--processed_root`
- `--recursive`

默认 `dataset_name` 仍为 `m3fd`，这是为了兼容旧用法；显式传入 `--raw-root` 后可以处理任意图像目录。

### 7. 训练入口解耦

`train.py` 已切换到 `build_sr_dataset()`。

改动点：

- `--dataset_cfg` 默认值改为 `configs/dataset/default.yaml`。
- 新增 `--dataset_name` / `--dataset-name`。
- 新增 `--raw_data_root` / `--raw-data-root`。
- 训练和验证 Dataset 构造时传入 `raw_root`。
- 日志新增 dataset cfg、dataset name、raw data root 信息。

### 8. 测试入口解耦

`test.py` 和 `engine/evaluator.py` 已切换到 `GenericSRDataset`。

改动点：

- `--dataset_cfg` 默认值改为 `configs/dataset/default.yaml`。
- 新增 `--dataset_name` / `--dataset-name`。
- 新增 `--raw_data_root` / `--raw-data-root`。
- evaluator 构造测试集时传入 `raw_root`。
- 单模型和多模型评估均使用同一通用数据集路径。

## 分 commit 建议

由于当前仓库 `.gitignore` 忽略整个 `datasets/` 目录，提交数据集源码改动时需要使用 `git add -f datasets/...`，或先调整 `.gitignore`。建议按以下提交拆分：

1. `refactor datasets to use generic SR dataset`
2. `add generic dataset builder API`
3. `add generic dataset config and registry`
4. `generalize dataset preprocessing`
5. `use generic dataset in training`
6. `use generic dataset in evaluation`
7. `add generic dataset unit tests`
8. `add M3FD decoupling report`

建议提交命令示例：

```bash
git add -f datasets/sr_dataset.py datasets/m3fd_dataset.py datasets/kaist_dataset.py
git commit -m "refactor datasets to use generic SR dataset"

git add -f datasets/builder.py datasets/__init__.py
git commit -m "add generic dataset builder API"

git add -f datasets/registry.py configs/dataset/default.yaml
git commit -m "add generic dataset config and registry"

git add -f datasets/preprocess.py
git commit -m "generalize dataset preprocessing"

git add train.py
git commit -m "use generic dataset in training"

git add test.py engine/evaluator.py
git commit -m "use generic dataset in evaluation"

git add tests/test_generic_dataset.py
git commit -m "add generic dataset unit tests"

git add 项目解耦.md
git commit -m "add M3FD decoupling report"
```

## 使用方式

### 使用任意未注册数据集

```bash
py main.py preprocess -- --raw-root D:/datasets/my_ir --recursive --overwrite
py main.py train -- --model edsr --scale 2 --dataset-cfg configs/dataset/default.yaml --raw-data-root D:/datasets/my_ir
py main.py test -- --model edsr --scale 2 --dataset-cfg configs/dataset/default.yaml --raw-data-root D:/datasets/my_ir
```

### 使用注册数据集默认值

```bash
py main.py preprocess -- --dataset-name m3fd --overwrite
py main.py train -- --model edsr --scale 2 --dataset-name m3fd
py main.py test -- --model edsr --scale 2 --dataset-name m3fd
```

## 注意事项

- `datasets/` 当前被 `.gitignore` 忽略，普通 `git status` 不会显示其中的新增和修改文件。
- `outputs/tmp_decouple` 为本次验证生成的临时数据和结果，位于已忽略的 `outputs/` 目录中。
- M3FD 兼容逻辑仍保留在 `GenericSRDataset` 的历史路径 remap 中，目的是保障旧 split 文件可继续使用，不影响新数据集接入。
